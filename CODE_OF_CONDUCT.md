# Code of Conduct

This project follows the spirit of the Contributor Covenant Code of Conduct. Everyone participating in `colbert-gguf-converter` is expected to help keep the project welcoming, focused, and useful.

## Our Standards

Examples of behavior that helps the project:

- being respectful, patient, and constructive
- focusing criticism on code, documentation, tests, and reproducible behavior
- accepting that open source software has rough edges without making reports personal
- giving clear technical context when disagreeing
- respecting maintainer decisions about project scope and release readiness

Examples of unacceptable behavior:

- harassment, threats, insults, or discriminatory language
- personal attacks or sustained off-topic disruption
- publishing private information without permission
- intentionally misleading correctness, performance, or compatibility claims
- repeated behavior that makes collaboration harder after maintainers ask for it to stop

## Scope

This code of conduct applies in project spaces, including issues, pull requests, discussions, reviews, release conversations, and other public or private project communication.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or restrict participation when behavior harms the project or community. Maintainers should apply these actions consistently and with the goal of keeping technical collaboration possible.

## Reporting

Report code-of-conduct concerns privately to the maintainers at security@mayflower.de. If the repository is published under an organization with a dedicated community contact, use that contact as the preferred reporting path.

Security vulnerabilities should follow [SECURITY.md](SECURITY.md), not public issue threads.
